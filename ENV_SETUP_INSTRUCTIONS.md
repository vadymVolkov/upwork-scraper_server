# Инструкция по настройке .env файла

## Шаг 1: Создайте файл .env

В корне проекта создайте файл с именем `.env` (с точкой в начале).

## Шаг 2: Перенесите данные из config.toml

Откройте файл `config.toml` и скопируйте следующие данные в файл `.env`:

### Формат для .env файла:

```env
# Upwork Credentials
UPWORK_USERNAME=volkov.vadym@gmail.com
UPWORK_PASSWORD=3V7a5D0y5M7!
```

### Важные правила:
1. **НЕ используйте кавычки** вокруг значений (в отличие от TOML)
2. **НЕ ставьте пробелы** вокруг знака `=`
3. Каждая переменная на отдельной строке
4. Можно добавлять комментарии, начиная строку с `#`

### Пример правильного формата:
```env
UPWORK_USERNAME=volkov.vadym@gmail.com
UPWORK_PASSWORD=3V7a5D0y5M7!
```

### Пример неправильного формата (НЕ ДЕЛАЙТЕ ТАК):
```env
UPWORK_USERNAME = "volkov.vadym@gmail.com"  ❌ (пробелы и кавычки)
UPWORK_PASSWORD='3V7a5D0y5M7!'              ❌ (кавычки)
```

## Шаг 3: Проверьте файл

После создания файл `.env` должен находиться в корне проекта рядом с `main.py` и `config.toml`.

## Готово!

После создания файла код автоматически будет использовать данные из `.env` вместо `config.toml` для credentials.

**Примечание:** Параметры поиска (Search) остаются в `config.toml`, так как они не являются чувствительными данными.
